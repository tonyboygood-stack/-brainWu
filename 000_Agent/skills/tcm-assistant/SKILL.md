# 中醫病案整理技能

## 觸發情境
使用者貼上門診筆記，或提到「整理病案」「問診紀錄」「今天的診」「處方」「診斷」「主訴」「病患」。
使用者可同時附上舌診照片，Claude 會讀取圖片輔助填入診斷內容。

---

## 圖片處理規則

使用者若附上照片（通常為舌診圖）：
1. 讀取圖片內容，輔助判斷舌象（舌色、舌苔、舌形等），填入「診斷」區塊
2. 自動建議圖片檔名：`YYYY-MM-DD_患者簡稱_舌診.jpg`
3. 在病案卡片內文加入圖片連結：`![舌診圖](../Assets/YYYY-MM-DD_患者簡稱_舌診.jpg)`
4. 存檔完成後提醒使用者：
```
請將舌診照片存入 400_Atlas/Assets/，
檔名：YYYY-MM-DD_患者簡稱_舌診.jpg
（在 Obsidian 手機版存入即可）
```

---

## 流程

### 第一步：自動填入五區塊

從使用者輸入的原始文字，自動抽取並填入以下格式：

```
## 主訴
[患者主要不適、就診原因、症狀描述]

## 診斷
[中醫辨證診斷；若有西醫診斷亦可並列]

## 處方
[藥方、穴位、針灸處置等；格式盡量保留原有結構]

## 追蹤重點
[下次複診需觀察的指標、警示症狀、生活建議]

## 疑問思考
[本案未確定的辨證方向、值得查閱的文獻、對治療選擇的自我提問]
```

**規則：**
- 能從輸入中抽取的就自動填入，不捏造內容
- 缺少的區塊標注「（本次紀錄未提及）」，並逐一詢問使用者是否補充
- 藥名、穴位名稱保留原文，劑量確實保留

### 第二步：顯示草稿，詢問確認

顯示完整五區塊後問：
```
這樣的整理有沒有需要補充或調整的地方？確認後我會存檔並建立疑問筆記。
```

### 第三步：存入病案卡片

使用者確認後，將病案存入：
```
400_Atlas/Case/YYYY-MM-DD_患者簡稱.md
```

病案卡片格式：
```markdown
---
created: YYYY-MM-DD
related:
  - "[[對應Questions筆記]]"
---

# YYYY-MM-DD 患者簡稱

## 主訴
...

## 診斷
...

## 處方
...

## 追蹤重點
...

## 疑問思考
...
```

### 第四步：疑問思考 → Questions 筆記

對「疑問思考」區塊中的每個問題，各自建立一張筆記存入 `400_Atlas/Questions/`：

```markdown
---
up: []
collection:
  - "[[Questions我好奇什麼]]"
related:
  - "[[對應病案檔名]]"
created: YYYY-MM-DD
says: "（問題本身）"
---

# 問題標題

（從疑問思考區塊抽取的原始問題）
```

同時在病案卡片的 `related` 欄位反向加入這些 Questions 筆記的連結。

---

## 注意事項
- 患者簡稱使用主訴關鍵字或症狀，不記錄真實姓名
- 疑問思考若為空，詢問使用者後跳過建立 Questions 筆記的步驟
- Questions 筆記的 `up` 欄位留空，等使用者日後整理時透過 note-capture 流程補上
