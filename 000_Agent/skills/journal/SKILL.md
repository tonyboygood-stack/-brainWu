# Learning Journal Skill — 老吳版

## 使用者背景
老吳是一位執業中醫師，寶寶剛出生四個月，年底準備自行開診所。時間極度破碎，需要最低阻力的記錄系統。他有正負兩個三年願景，每次寫日記都是在為其中一個投票。

願景檔案路徑：`C:/Users/user/vision.md`（閱讀時作為 Q3 的參考素材）

---

## 對話開頭閉環檢查

**每次新對話開始時**，執行以下步驟：

1. 計算昨天的日期（today - 1 day），格式 `YYYY-MM-DD`
2. 嘗試讀取 `300_Journal/YYYY-MM/YYYY-MM-DD.md`
3. 搜尋「明天待辦」與「明天的期待」區塊
4. 如果找到內容，在回應開頭加上：

```
早安！昨天你為明天準備了這些：

【待辦】
- [ ] 事項一
- [ ] 事項二

【明天的期待】
- 讓明天更有趣的事

要從哪個開始？
```

5. 如果沒有昨日日記或沒有內容，靜默略過，正常回應。

---

## 寫日記流程

觸發後，**先從當前對話整理今日內容**，再依序逐一問問題（等使用者回答後再問下一題）：

### 步驟 0 — 從當前對話整理今日內容

1. 回顧這段對話中老吳今天提到的所有事情（工作、看診、學習、生活、想法等）
2. 整理成條列，顯示給使用者：

```
根據我們今天的對話，你今天做了這些事：

- 事項 A
- 事項 B
- 事項 C

有漏掉的嗎？可以補充，或直接說「沒有」繼續。
```

3. 使用者補充完後，將整理內容 + 補充合併為今日活動素材，**直接進入問題 2**（跳過問題 1）。
4. **若對話中完全沒有今日相關內容**，才直接問問題 1。

---

### 問題 1 — 今天做了什麼（對話中無素材時才問）
```
今天主要做了什麼？（工作、學習、生活都可以，口語就好）
```

---

### 問題 2 — 今天學到什麼
```
今天有什麼新發現、新理解、或讓你印象深刻的事嗎？
```

---

### 問題 3 — 卡住 × 願景對齊

**先問卡住的地方：**
```
今天有沒有卡住、做不順、或想改善的地方？
```

使用者回答後，**讀取 `C:/Users/user/vision.md` 的正向願景**，從中挑選 2-3 個與今天情境最相關的具體畫面，問：

```
換個角度——今天有沒有哪個時刻，其實是在為你想要的那個未來投票？

比如：
• 下診後真的坐下來寫了幾行觀察？（臨床記錄習慣）
• 蹲下來認真聽孩子說話？（家庭陪伴）
• 選擇早睡而不是滑手機？（身體照顧）

（不一定要完美，小小的一個選擇就算）
```

> **注意**：括號內的例子要根據 vision.md 和當天 Q1/Q2 的內容動態替換，選最貼近今天的 2-3 個，不要每次都一樣。

---

### 問題 4 — 明天待辦
```
明天（或接下來）你最想完成的 1-3 件事是什麼？
```

---

### 問題 4b — 明天的期待
```
明天有什麼讓你期待的、或可以讓明天更有趣精彩的事？（哪怕很小都算）
```

> 這個答案和明天待辦一樣，都會出現在隔天對話開頭的提醒中。

---

### 問題 5 — 今天的心情或一句話
```
用一句話或一個詞形容今天的狀態？
```

---

### 人物側寫（Q5 結束後執行）

收集完所有回答後，在生成日記之前，先寫一段**人物側寫**：

用**小說第三人稱筆法**，以「老吳」為主角，結合今天的 QA 內容，描寫他在這個生命階段（中醫師、新手爸爸、準備開診所）裡，今天如何努力前進、活出自己。

**風格要求**：
- 300 字以內，文字有溫度、有細節、有真實感
- 不誇張，不勵志格言，就是平實的一天
- 帶出老吳正在為哪個未來投票的感覺
- 可以有光，也可以有疲憊，但要讓人感覺這個人值得被記錄

**範例風格（僅供參考，內容每天不同）**：
> 老吳今天下診比預期晚了半小時。最後一個病人的脈象有些複雜，他沒有草草帶過，而是多問了幾句——不是因為診所要求，而是職業本能讓他停在那個當下。回家的路上，他在手機備忘錄裡打了幾個字，等孩子睡了再整理。寶寶今天特別黏他，他把孩子扛在肩上繞了幾圈，什麼都沒說，就是走著。那幾分鐘，好像什麼都停下來了。

---

## 生成日記

人物側寫完成後，生成完整日記格式，**一次顯示給使用者確認**：

```markdown
# 📓 YYYY-MM-DD 學習日記

## 今天做了什麼
（整理使用者的回答，保留口語感，精簡通順）

## 今天學到什麼
（整理學習心得，可加一句延伸觀察）

## 卡住 / 想改善
（整理困難點，語氣中性）

## 今天為哪個未來投票
（整理 Q3 願景對齊的回答，1-2 句，不需要完美）

## 明天待辦
- [ ] 事項一
- [ ] 事項二
- [ ] 事項三

## 明天的期待
- 讓明天更有趣的事

## 今日狀態
> 使用者的一句話或詞

---

## 人物側寫
（剛才生成的第三人稱小說段落）

---
*記錄於 HH:MM*
```

詢問使用者：
```
這樣可以嗎？確認後我就存起來。（也可以說「修改：...」讓我調整）
```

---

## 儲存日記

使用者確認（說「好」「可以」「存」「沒問題」等）後：

1. 確保 `300_Journal/YYYY-MM/` 目錄存在（不存在則建立）
2. 將完整日記（含人物側寫）寫入 `300_Journal/YYYY-MM/YYYY-MM-DD.md`
3. 回應：

```
已存到 300_Journal/YYYY-MM/YYYY-MM-DD.md ✓
明天開對話時我會提醒你待辦和明天的期待。晚安！
```

---

## 注意事項

- **逐題問**：每次只問一個問題，等使用者回答再繼續。
- **保留口語**：整理答案時保留老吳的語氣，不要文謅謅。
- **日期以系統時間為準**：使用 `date` 指令獲取當前日期。
- **儲存路徑**：`300_Journal/YYYY-MM/` 相對於使用者當前工作目錄（`pwd`）。
- **人物側寫每天都不同**：根據當天 QA 內容即興生成，不要套模板。
- **Q3 願景例子要動態替換**：從 vision.md 讀取後，挑今天最相關的 2-3 個，不重複。
- **閉環是可選的**：如果昨日無日記，靜默略過。
