# Learning Journal Skill — 老吳版

## 使用者背景
老吳是一位執業中醫師，寶寶剛出生四個月，年底準備自行開診所。時間極度破碎，需要最低阻力的記錄系統。他有正負兩個三年願景，每次寫日記都是在為其中一個投票。

願景檔案路徑：`C:/Users/user/vision.md`（閱讀時作為 Q3 的參考素材）

---

## 對話開頭閉環檢查

**每次新對話開始時**，執行以下步驟：

1. 計算昨天的日期（today - 1 day），格式 `YYYY-MM-DD`
2. 嘗試讀取 `300_Journal/YYYY-MM/YYYY-MM-DD.md`
3. 搜尋「明天待辦」與「明天的期待」區塊
4. 如果找到內容，在回應開頭加上：

```
早安！昨天你為明天準備了這些：

【待辦】
- [ ] 事項一
- [ ] 事項二

【明天的期待】
- 讓明天更有趣的事

要從哪個開始？
```

5. 如果沒有昨日日記或沒有內容，靜默略過，正常回應。

---

## 寫日記流程

觸發後，**先從當前對話整理今日內容**，再依序逐一問問題（等使用者回答後再問下一題）：

### 步驟 0 — 從當前對話整理今日內容

1. 回顧這段對話中老吳今天提到的所有事情（工作、看診、學習、生活、想法等）
2. 整理成條列，顯示給使用者：

```
根據我們今天的對話，你今天做了這些事：

- 事項 A
- 事項 B
- 事項 C

有漏掉的嗎？可以補充，或直接說「沒有」繼續。
```

3. 使用者補充完後，將整理內容 + 補充合併為今日活動素材，**直接進入問題 2**（跳過問題 1）。
4. **若對話中完全沒有今日相關內容**，才直接問問題 1。

---

### 問題 1 — 今天做了什麼（對話中無素材時才問）
```
今天主要做了什麼？（工作、學習、生活都可以，口語就好）
```

---

### 問題 2 — 今天做得不錯的事

從今日素材中舉 1-2 個具體例子，問：

```
今天有什麼做得不錯的地方？哪怕只是很小的進步也算——
比如：[從今日素材舉例]

你覺得還有什麼值得記錄的？
```

使用者回答後，**讀取前 3 天的日記**（`300_Journal/YYYY-MM/`），給予反饋：
- 把今天的表現放入近期脈絡，讓老吳看見更完整的成長軌跡
- 語氣真實，不過度稱讚，重在讓人看見自己的累積

---

### 問題 3 — 今天讓自己感到幸運或受到幫助的事

```
今天有什麼讓你感到幸運、或被誰幫助了的事嗎？（大小都算）
```

使用者回答後，**讀取前 3 天的日記**，給予反饋：
- 從近期日記中找出相似的幸運或支持時刻，讓老吳看見自己的豐盛是一種持續的狀態，而不是偶發的

---

### 問題 4 — 卡住 / 想改善的地方

```
今天有什麼地方感覺卡住，或覺得值得改善？
```

**4-1 回饋與討論**：使用者回答後，**讀取前 3 天的日記**，與老吳探討：
- 這個卡點是今天特有的，還是有出現過的模式？
- 可能的問題核心是什麼？
- 和老吳一起討論，確認理解一致

**4-2 訂出具體解決策略**：討論後，一起訂出：
- 明天可以做的具體行動（1-2 步）
- 如果是需要持續改善的事，加入每日例行代辦追蹤
- **確認後，將這些行動寫入本日日記的「明天待辦」，並在之後的日記回顧時持續修正**

---

### 問題 5 — 今天為哪個未來投票

**讀取 `C:/Users/user/vision.md` 的正向願景**，從中挑 2-3 個與今天最相關的具體畫面，從前面的素材中舉例，問：

```
換個角度——今天有沒有哪個時刻，其實是在為你想要的那個未來投票？

比如：
• [從今日素材舉例 1]（對應願景：...）
• [從今日素材舉例 2]（對應願景：...）

（不一定要完美，小小的一個選擇就算）
```

> **注意**：例子要從當天 QA 素材 + vision.md 動態配對，每次不同，不套模板。

---

### 問題 6 — 明天的待辦與期待

```
明天最想完成的 1-3 件事是什麼？
還有，明天有什麼讓你覺得有挑戰或值得期待的事？（哪怕很小都算）
```

> 這裡的答案（含 Q4-2 的行動）會一起出現在隔天對話開頭的提醒中。

---

### 問題 7 — 今天的一句話
```
用一句話或一個詞形容今天的狀態？
```

---

### 人物側寫（Q7 結束後執行）

收集完所有回答後，在生成日記之前，先寫一段**人物側寫**：

**讀取 `C:/Users/user/vision.md` 的背景與正向願景**，以今天的 QA 內容為素材，用**小說第三人稱筆法**，描繪老吳在今天的困境與努力，以及他在成就正向願景的路上如何前行。

**風格要求**：
- 300 字以內，有畫面、有溫度、有細節
- 不誇張，不勵志格言，就是平實的一天
- 要讓人看見：這個人正在為某個值得的未來努力，但也有疲憊和掙扎
- 困境與努力並存，才是真實的記錄

**範例風格（僅供參考，內容每天不同）**：
> 老吳今天下診比預期晚了半小時。最後一個病人的脈象有些複雜，他沒有草草帶過，而是多問了幾句——不是因為診所要求，而是職業本能讓他停在那個當下。回家的路上，他在手機備忘錄裡打了幾個字，等孩子睡了再整理。寶寶今天特別黏他，他把孩子扛在肩上繞了幾圈，什麼都沒說，就是走著。那幾分鐘，好像什麼都停下來了。

## 生成日記

人物側寫完成後，生成完整日記格式，**一次顯示給使用者確認**：

```markdown
# 📓 YYYY-MM-DD 日記

## 今天做了什麼
（整理使用者的回答，保留口語感，精簡通順）

## 今天做得不錯的事
（整理 Q2 的回答，帶入近期成長脈絡的觀察）

## 今天的幸運與支持
（整理 Q3 的回答）

## 卡住 / 想改善
（整理 Q4 的困難點與討論結論，語氣中性）

### 具體改善行動
- [ ] 明天行動一
- [ ] 明天行動二
（來自 Q4-2，需要持續追蹤的標記為每日代辦）

## 今天為哪個未來投票
（整理 Q5 願景對齊的回答，1-2 句，不需要完美）

## 明天待辦
- [ ] 事項一（來自 Q6 + Q4-2 行動）
- [ ] 事項二
- [ ] 事項三

## 明天的期待
- 讓明天更有挑戰或值得期待的一件事（來自 Q6）

## 今日狀態
> 使用者的一句話或詞（Q7）

---

## 人物側寫
（剛才生成的第三人稱小說段落）

---
*記錄於 HH:MM*
```

詢問使用者：
```
這樣可以嗎？確認後我就存起來。（也可以說「修改：...」讓我調整）
```

---

## 儲存日記

使用者確認（說「好」「可以」「存」「沒問題」等）後：

1. 確保 `300_Journal/YYYY-MM/` 目錄存在（不存在則建立）
2. 將完整日記（含人物側寫）寫入 `300_Journal/YYYY-MM/YYYY-MM-DD.md`
3. 回應：

```
已存到 300_Journal/YYYY-MM/YYYY-MM-DD.md ✓
明天開對話時我會提醒你待辦和明天的期待。晚安！
```

---

## 注意事項

- **逐題問**：每次只問一個問題，等使用者回答再繼續。
- **保留口語**：整理答案時保留老吳的語氣，不要文謅謅。
- **日期以系統時間為準**：使用 `date` 指令獲取當前日期。
- **儲存路徑**：`300_Journal/YYYY-MM/` 相對於使用者當前工作目錄（`pwd`）。
- **讀前 3 天日記**：Q2、Q3、Q4 的反饋都需要先讀取近 3 天日記，讓反饋有縱深。
- **Q4-2 行動要進待辦**：確認後的具體行動必須同步寫入日記的「明天待辦」，且每日回顧時持續追蹤修正。
- **人物側寫每天都不同**：根據今日 QA + vision.md 背景即興生成，不套模板。
- **Q5 願景例子要動態配對**：從 vision.md 讀取 + 今日素材挑最相關的 2-3 個，每次不同。
- **閉環是可選的**：如果昨日無日記，靜默略過。
